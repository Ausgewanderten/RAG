# WavEDA 操作教学型 Agent 知识库

## 0. 文档定位

本文档用于构建一个面向小白用户的 **WavEDA 操作教学型 Agent**。  
目标是帮助完全不熟悉 WavEDA 的用户完成电磁 / 光电仿真建模、参数设置、报错排查、结果查看、扫频计算和后处理对比。

本知识库不按单个历史案例作为主体分类，而是按以下两条主线组织：

1. **仿真生命周期**  
   从拿到模型，到建模、材料、边界、端口 / 光源、网格、单频、扫频、后处理、对比验证。

2. **操作问题域**  
   按几何、材料、端口、边界、网格、监视器、S 参数、报错、数据导出等问题域整理。

历史案例，如 03 定向耦合器、09 直栅型光栅耦合器、012 strip-to-slot 模式转换器、Ref-A 微波结构，只作为附录示例，不作为主分类依据。

---

# 1. Agent 总体工作原则

## 1.1 核心原则

WavEDA 教学型 Agent 不应该一上来就让用户跑完整仿真，而应该遵循：

> **先判断模型类型 → 再统一物理口径 → 单频验证 → 再全频扫频 → 最后后处理对比。**

对于小白用户，最容易出错的不是按钮不会点，而是以下几类问题：

- 不知道当前仿真属于哪种物理问题；
- 把端口网络问题和光源照射问题混为一谈；
- 材料、单位、边界、光源和后处理口径没有统一；
- 一开始就全频扫频，导致耗时长且无法定位错误；
- 端口模式数、端口方向、S 参数通道读错；
- WavEDA 和 COMSOL / EastWave / 参考软件结果不一致时，不知道先查哪里。

因此 Agent 应该始终引导用户按步骤推进，而不是直接给一堆参数。

## 1.2 推荐对话节奏

当用户说“我不知道怎么设”“这个报错咋办”“接下来怎么做”时，Agent 应该优先执行：

1. **确认当前阶段**
   - 是刚导入模型？
   - 还是已经设完材料？
   - 是卡在端口 / 光源？
   - 是卡在网格？
   - 是已经跑完单频？
   - 是扫频后处理？

2. **判断仿真类型**
   - 微波 S 参数端口网络？
   - 集成光波导端口网络？
   - 光栅耦合器光源照射？
   - 周期 / Floquet / 平面波入射问题？
   - FDTD 瞬态问题？
   - 频域 FEM / EWBE 问题？

3. **让用户优先给关键截图**
   - 几何整体截图；
   - 材料列表；
   - 边界设置；
   - 端口 / 光源设置；
   - 求解器设置；
   - 报错完整文本；
   - 单频结果图；
   - S 参数曲线或导出数据。

4. **先做最小闭环**
   - 不要直接全频扫；
   - 先跑一个代表频点；
   - 先看端口模态、场图、S 参数是否合理；
   - 合理后再扫频。

# 2. 仿真类型快速判别

## 2.1 端口网络型仿真

典型对象：

- 微波四端口结构；
- 定向耦合器；
- strip-to-slot 模式转换器；
- 普通波导器件；
- 多端口 S 参数器件。

核心特征：

- 有明确输入端口和输出端口；
- 目标是得到 `S11 / S21 / S31 / S41` 等；
- 一般需要设置 Wave Port / Mode Port / Network Parameter；
- 一次通常只激励一个端口；
- 重点检查端口编号、端口方向、端口模式数和 S 参数通道。

典型输出：

- S 参数曲线；
- 端口模态图；
- 电场 / 磁场分布；
- SNP 文件；
- 与 COMSOL / EastWave 的 S 参数误差对比。

## 2.2 光源照射型仿真

典型对象：

- 直栅型光栅耦合器；
- 垂直 / 斜入射光耦合结构；
- 平面波 / 高斯光束照射问题。

核心特征：

- 输入不是普通波导端口，而是空间光源；
- 输出可能是波导截面功率；
- 不一定直接得到标准多端口 S 参数；
- 重点不是“Port1/Port2/Port3”，而是：
  - 光源类型；
  - 入射角；
  - 偏振；
  - 光斑位置；
  - 接收截面；
  - 能流积分。

典型输出：

- 输出波导截面功率；
- `Sx / Sy / Sz` 能流；
- 场分布；
- 耦合效率；
- 波长 / 频率响应曲线。

## 2.3 周期 / Floquet / 平面波入射型仿真

典型对象：

- 周期性超表面；
- FSS；
- Ref-A / ref’-A 类微波周期结构；
- 无地 / 有地结构；
- 带入射角的平面波问题。

核心特征：

- 需要周期边界；
- 需要 Floquet 端口或平面波；
- 只设置 `kFloquet` 不一定够，通常还要设置端口入射角；
- 仰角 / 方位角 / 极化方向非常关键；
- 端口设置错会导致 S 参数完全不对。

典型输出：

- 反射 / 透射 S 参数；
- TE / TM 模式响应；
- 不同入射角下曲线；
- 与参考 PPT 曲线对比。

# 3. 仿真生命周期总流程

## 3.1 生命周期总览

推荐 Agent 采用以下流程指导用户：

```text
明确任务目标
    ↓
判断仿真类型
    ↓
检查几何与单位
    ↓
设置材料
    ↓
设置计算区域 / 空气盒 / 包层
    ↓
设置边界条件
    ↓
设置端口 / 光源
    ↓
设置监视器 / Snapshot
    ↓
设置网格
    ↓
单频点试跑
    ↓
检查端口模态 / 场图 / S 参数
    ↓
全频扫频
    ↓
导出数据
    ↓
后处理对比
    ↓
误差分析与报告整理
```

Agent 不应跳过单频验证。

# 4. 阶段一：明确任务目标

## 4.1 Agent 首先要问清楚什么

对小白用户，Agent 要先确认：

1. 你现在做的是哪类模型？
   - 微波器件？
   - 光波导器件？
   - 光栅耦合器？
   - 模式转换器？
   - 周期结构？

2. 你要看什么结果？
   - S 参数？
   - 透射率 / 反射率？
   - 耦合效率？
   - 电场图？
   - 磁场图？
   - 端口模态？
   - 截面功率？

3. 你准备用什么求解方式？
   - 频域？
   - FDTD？
   - Network Parameter？
   - Floquet / 周期仿真？

4. 参考对象是什么？
   - COMSOL？
   - EastWave？
   - PPT 曲线？
   - 论文图？
   - 老板给的数据？

## 4.2 判断任务目标的模板

Agent 可以使用以下判断模板：

```text
你这个模型先不要急着跑全频。我们先判断它属于哪种仿真：

1. 如果输入输出都是波导端口，那它是端口网络问题，重点是端口方向、模式数和 S 参数通道。
2. 如果输入是上方斜入射光，那它不是普通三端口问题，而是光源照射 + 输出截面功率问题。
3. 如果结构是周期阵列或 FSS，那要重点检查周期边界、Floquet 端口、入射角和极化。
4. 如果你要和 COMSOL 对比，必须先统一材料、频率范围、边界、端口激励和后处理公式。
```

# 5. 阶段二：几何与单位检查

## 5.1 几何检查优先级

WavEDA 仿真前，Agent 应指导用户检查：

1. 模型是否完整；
2. 单位是否正确；
3. 空气盒 / 包层是否包住器件；
4. 端口截面是否露出来；
5. 是否存在不连续、缝隙、重叠、零厚度面；
6. 导入 STL / STEP 后是否变成不可编辑网格；
7. 是否需要手搓原生几何，而不是直接依赖导入网格。

## 5.2 光波导几何注意点

集成光波导器件常见材料层：

```text
空气上包层
Si 芯层
SiO2 BOX / 下包层
必要时还有 Si substrate
```

常见错误：

- 忘记空气上包层；
- SiO2 相对介电常数设错；
- 只导入硅结构，没有背景包层；
- 空气盒没有完整包住结构；
- 端口没有覆盖完整波导截面；
- slot waveguide 的输出端口只选了一条硅条，没有覆盖整个 slot 截面。

## 5.3 微波几何注意点

微波结构常见要点：

- 基板厚度；
- 金属铜层厚度；
- PEC 地；
- 端口高度是否包含铜厚；
- 空气域大小；
- 端口是否贴在正确截面；
- 周期结构是否保留完整单元。

对于有地结构：

- 必须确认 PEC Ground 的位置；
- 地平面不能误删；
- 有地 / 无地模型不能混用边界和端口设置。

## 5.4 导入模型 vs 原生建模

Agent 应优先建议：

- 如果只是看形状，可导入 STL / STEP；
- 如果要精确仿真，尤其是光波导器件，最好使用原生几何或参数化建模；
- 对定向耦合器、模式转换器、光栅等结构，原生 CAD / 参数化几何更可靠；
- 导入 STL 可能出现：
  - 非流形面；
  - 缝隙；
  - 重叠；
  - 端口无法识别；
  - 网格异常；
  - 材料赋值不稳定。

# 6. 阶段三：材料设置

## 6.1 材料设置总原则

材料必须和参考软件统一，否则 S 参数和场图很容易偏差。

Agent 应要求用户确认：

```text
材料名称
相对介电常数 εr
折射率 n
损耗角正切 tanδ
电导率
材料作用区域
频率相关性
是否与 COMSOL / EastWave / 脚本一致
```

## 6.2 光波导常用材料

常见设置：

```text
Si:
n ≈ 3.45
εr ≈ 11.90

SiO2:
n ≈ 1.44
εr ≈ 2.07

Air:
n = 1
εr = 1
```

历史踩坑：

- SiO2 曾被误设为 `εr = 4.2`，导致模式转换器结果明显不对；
- 修正为 `εr ≈ 2.07` 后，WavEDA 的 `S21 / S12` 与 COMSOL 基本一致；
- EastWave 文档没有明确写空气上包层，但 Python 脚本里体现了空气背景，因此 WavEDA 也应补上空气域。

## 6.3 微波常用材料

Ref-A 类微波结构曾涉及：

```text
RO5430B:
εr = 3.66
tanδ = 0.0037

Copper:
厚度约 0.035 mm

PEC Ground:
用于有地结构
```

注意：

- 铜厚是否参与端口高度要确认；
- PEC Ground 和普通铜层不能混淆；
- 有地模型和无地模型的边界条件不同。

## 6.4 材料排查模板

当结果明显不对时，Agent 可以先让用户查材料：

```text
先别急着改网格。S 参数整体偏移或 S21 明显不对，第一优先级通常是材料口径。
你检查这几项：

1. Si 的 εr / n 是否和参考一致；
2. SiO2 是否误设成 4.2；
3. 空气上包层是否存在；
4. 背景材料到底是空气还是 SiO2；
5. 金属是铜、PEC 还是有限电导率；
6. 损耗有没有打开；
7. 单位是 GHz 还是 THz。
```

# 7. 阶段四：边界条件与计算区域

## 7.1 边界设置总原则

边界的作用是模拟开放空间、周期结构、金属壁或吸收层。

常见边界类型：

```text
默认吸收边界
PML / CPML
散射边界
周期边界
PEC
PMC
Floquet 周期边界
```

## 7.2 WavEDA 与 COMSOL 边界差异

经验规律：

- COMSOL 中很多开放域需要用户手动设置空气盒、散射边界或 PML；
- WavEDA 有时自带默认吸收边界，不一定需要用户额外手动加 PML；
- 某些 WavEDA 波端口工作流中，显式 PML 和波端口可能不兼容；
- 当出现“波端口与 PML 不兼容”类问题时，优先检查是否重复设置了 PML 或边界类型选错。

## 7.3 光波导边界建议

对于集成光波导：

```text
横向和上下方向：
使用开放 / 吸收 / PML 类边界

传播方向：
端口所在截面使用端口边界
非端口外边界使用开放 / 吸收边界
```

注意：

- 空气盒或包层必须足够大；
- PML 不要贴得太近；
- 端口面不要被 PML 覆盖；
- 端口面应和波导传播方向垂直；
- 端口要覆盖完整模式截面，而不只是硅芯本身的一小块。

## 7.4 微波边界建议

对于微波端口网络：

- 有地结构需要确认 PEC Ground；
- 四周开放域可用吸收边界；
- 周期结构需要周期 / Floquet；
- 端口面不能和 PML 冲突；
- 端口所在面要完整覆盖传输截面。

## 7.5 周期 / Floquet 边界建议

对于周期结构：

1. 周期方向两侧设置周期边界；
2. 入射方向设置 Floquet 端口或平面波；
3. 不仅要设置 `kFloquet`，还要检查：
   - 仰角；
   - 方位角；
   - 极化；
   - 端口模式；
   - 入射方向；
   - 端口法向。

历史踩坑：

- 只设置 `kFloquet` 不够；
- 端口 2 的角度需要根据周期性端口和折射率关系确认；
- 端口编号和方向混淆会导致 S 参数通道错误。

# 8. 阶段五：端口与光源设置

端口 / 光源是 WavEDA 最容易出错的部分。  
Agent 要先判断：当前模型到底需要“端口”还是“光源”。

## 8.1 端口网络问题

适用对象：

- 定向耦合器；
- 模式转换器；
- 微波多端口器件；
- 普通波导器件。

核心检查项：

```text
端口数量
端口编号
端口位置
端口方向
端口截面大小
激励端口
非激励端口
端口模式数
TE / TM / 主模
S 参数通道对应关系
```

## 8.2 一次只激励一个端口

对于标准 S 参数仿真：

> 通常一次只激励一个端口，其余端口作为接收 / 匹配端口。

常见错误：

- 同时激励多个端口，导致场相互抵消；
- 以为设置了 4 个端口就会自动得到所有 S 参数；
- COMSOL 中没有开启 Port Sweep 或只算了端口 1 激励；
- WavEDA 中激励端口和输出端口方向弄反；
- 读 S 参数时把模式 2 当成模式 1。

## 8.3 端口方向判断

端口方向不能凭感觉，要看端口箭头和传播方向。

通用原则：

```text
输入端口：
能量应该从端口进入计算域

输出端口：
能量应该从计算域传播到端口截面
```

对于沿 x 方向传播的器件：

```text
左侧输入端口：
通常从左向右进入器件

右侧输出端口：
通常从左向右离开器件
```

但 WavEDA 的端口方向可能用 forward / backward 或端口法向表示，因此必须看界面箭头。

Agent 回答模板：

```text
不要只看它在左边还是右边，要看端口箭头。
输入端口的箭头应该表示能量进计算域；输出端口虽然是“接收”，但端口本身也要能正确识别从器件出来的模式。
如果 S21 很小但场明显传过去了，优先怀疑端口方向或 S 参数通道读错。
```

## 8.4 端口截面大小

### 光波导端口

端口不应只选硅芯的一个小面，而应覆盖完整模式截面。

例如：

- strip waveguide 输入端口：覆盖输入波导及周围包层区域；
- slot waveguide 输出端口：覆盖整个 slot waveguide 模式截面，包括两条硅条、中间 slot 和周围包层；
- 三端口定向耦合器：Port1 输入，Port2 through，Port3 coupled，各端口位置与波导截面对应。

常见错误：

- 只选硅面，模式场被截断；
- slot 输出端口只覆盖一条硅条；
- 端口贴在结构内部或离结构太近；
- 端口面没有垂直于传播方向；
- 端口截面和计算域边界重合关系错误。

## 8.5 端口模式数

端口模式数非常关键。

经验：

- 简单单模波导可以先设置 1 个模式；
- 如果仿真结果异常、出现 mode2、或者参考软件有多个模式，需要设置 2 个或更多模式；
- 但增加模式后，必须明确读取的是哪个模式通道。

常见错误：

- 端口算出了 mode2，以为结果错误；
- 实际主模在 mode1，却读取了 mode2 的 S 参数；
- 端口模式顺序和 COMSOL 不一致；
- TE/TM 模式混淆。

排查建议：

```text
先看端口模态图。
主模应该集中在波导芯层或 slot 区域，场型要符合预期。
如果 mode1 场型不对，而 mode2 场型像目标模式，就需要重新确认 S 参数通道。
```

## 8.6 光源照射问题

适用对象：

- 光栅耦合器；
- 上方斜入射光；
- 高斯光源 / 平面波入射；
- FDTD 空间光源问题。

关键区别：

> 光栅耦合器不是普通三端口定向耦合器，不能强行按 Port1 / Port2 / Port3 的方式理解。

核心设置项：

```text
光源类型
光源位置
入射方向
入射角
偏振
中心频率
频率范围
光斑大小
输出监视器
功率积分方式
```

## 8.7 高斯光源缺失时的替代方案

历史经验：

- 某些 WavEDA 设置中没有和 COMSOL 完全一致的高斯面光源；
- 可暂时使用平面波替代；
- 但需要在报告中说明：光源口径不同，耦合效率绝对值可能不完全一致，趋势更有参考意义。

光栅耦合器历史近似参数：

```text
中心频率：
193.41 THz

坐标系：
笛卡尔坐标系

近似入射方向：
kx = -0.117
ky = 0
kz = -0.993

偏振：
线性极化
H(TE) = 1
V(TM) = 0
```

注意：

- 这是历史案例中的近似，不应无脑套用到所有模型；
- 新模型要根据入射角重新计算方向向量；
- 平面波不能完全等价于高斯光束。

# 9. 阶段六：监视器与 Snapshot 设置

## 9.1 监视器的作用

监视器用于查看：

- 电场；
- 磁场；
- 功率流；
- 截面场；
- 端口模态；
- 输出功率；
- 时间域 / 频域场分布。

## 9.2 常用监视器类型

| 需求 | 推荐监视器 |
|---|---|
| 看 xy 平面电场 | 面 Snapshot / Field Monitor |
| 看波导中间截面场 | z 固定的 XY 面 |
| 看输出功率 | 输出截面能流监视器 |
| 看端口模式 | Port Mode / Mode Profile |
| 看沿传播方向变化 | 线监视器 / 截面序列 |
| 看光栅耦合效率 | 输出波导截面功率积分 |

## 9.3 光波导常用场图

对于集成光波导，常看：

```text
E(abs)
Ey
Ex
Ez
H(abs)
Poynting vector
Sx / Sy / Sz
```

历史经验：

- 若想看 xy 面场图，可设置固定 z 的面快照；
- strip-to-slot 模式转换器曾使用 `z = 0.11 μm` 附近观察 XOY 面场；
- 如果输出沿 x 方向传播，功率流重点看 `Sx`；
- 若传播方向相反，可能要看 `-Sx`。

## 9.4 光栅耦合器输出监视器

直栅型光栅耦合器的关键不是简单 S21，而是输出波导截面功率。

典型思路：

```text
上方光源照射光栅
    ↓
光被耦合进侧向波导
    ↓
在输出波导截面放置功率监视器
    ↓
积分 Sx 或 -Sx 得到输出功率
    ↓
归一化得到耦合效率
```

若界面里没有“平面接收器”，可以改用：

- 面快照；
- 截面场监视器；
- 线监视器；
- 后处理积分；
- 观察 `Ey + mag`、`E(abs)` 或 `Sx`。

# 10. 阶段七：网格设置

## 10.1 网格设置原则

网格不是越细越好。  
Agent 应先让用户使用粗网格做单频验证，再逐步加密。

推荐顺序：

```text
粗网格单频
    ↓
检查端口 / 场图 / S 参数
    ↓
中等网格单频
    ↓
对比差异
    ↓
确认合理后再全频扫频
```

## 10.2 光波导网格重点区域

光波导需要加密的区域：

- Si / SiO2 界面；
- slot 区域；
- taper 区域；
- SWG / 光栅细结构；
- 耦合间隙；
- 端口截面附近；
- 高折射率突变区域。

常见错误：

- 全局网格过粗，slot 结构无法解析；
- 光栅周期结构网格不足；
- taper 尖端出现畸形网格；
- 只加密空气区，没加密硅芯；
- 自适应网格卡住；
- 单频点迭代次数达到上限仍未收敛。

## 10.3 微波网格重点区域

微波结构需要加密的区域：

- 金属边缘；
- 缝隙；
- 端口附近；
- 介质 / 金属交界；
- 周期结构边界；
- 强耦合区域。

## 10.4 网格相关报错处理

### 报错：网格卡住 / 太慢

排查顺序：

1. 几何是否过大；
2. 是否空气盒过大；
3. 是否细小结构太多；
4. 是否全局网格过细；
5. 是否自适应开启导致反复迭代；
6. 是否导入模型存在坏面；
7. 是否可以裁剪计算区域；
8. 是否可以先降阶 / 降低精度单频测试。

### 报错：迭代到最大次数未达到误差阈值

常见原因：

- 端口激励不正确；
- 端口模式不稳定；
- 边界反射强；
- 网格太粗；
- 材料或单位错误；
- 计算区域太小；
- 高 Q / 强谐振结构本身难收敛。

处理建议：

```text
不要立刻全局加密。
先检查端口方向、端口模式、边界距离、材料单位。
如果这些都没问题，再逐步提高网格精度或放宽误差阈值做预验证。
```

# 11. 阶段八：单频验证

## 11.1 为什么必须先单频

单频验证用于确认：

- 端口是否能正常激励；
- 场是否沿预期方向传播；
- 输出端口是否接收到能量；
- S 参数数量是否完整；
- 结果是否大致合理；
- 模型是否会报错；
- 仿真耗时是否可接受。

小白常见错误是直接扫 101 个点，结果跑很久后才发现端口设错。

## 11.2 单频点选择

### 光通信波段

常用中心频率：

```text
λ = 1550 nm
f ≈ 193.414 THz
```

可用作单频验证点：

```text
193.41 THz
193.616 THz
```

具体以参考模型为准。

### 微波频段

Ref-A 类历史模型：

```text
频段：2–8 GHz
单频验证点：2 GHz
```

## 11.3 单频后必须看什么

单频跑完后，不要只看 S 参数。  
Agent 应要求用户至少检查：

1. 端口模态图；
2. 电场分布；
3. 能量传播方向；
4. S11 是否异常大；
5. S21 / S31 是否有合理输出；
6. 是否有非物理反射；
7. 端口模式编号是否对应；
8. 是否有警告或收敛失败。

## 11.4 单频结果判断模板

```text
你这次单频结果先看三件事：

1. 场有没有从输入端口进来，并沿着器件传播；
2. 输出端口附近有没有明显场分布；
3. S 参数有没有数量级异常，比如 S21 接近 -100 dB、S11 接近 0 dB，但场图又显示能量传过去。

如果场图合理但 S 参数不合理，优先怀疑端口方向、模式数或 S 参数通道。
如果场图本身就不传播，优先查材料、光源、边界和端口激励。
```

# 13. 阶段十：S 参数与数据导出

## 13.1 S 参数基础

常见 S 参数含义：

```text
S11：输入端口反射
S21：从 Port1 到 Port2 的传输
S31：从 Port1 到 Port3 的耦合 / 传输
S41：从 Port1 到 Port4 的传输
```

对于三端口定向耦合器：

```text
Port1：输入
Port2：Through
Port3：Coupled
```

但这只是历史案例中的端口约定，新模型要根据几何重新确认。

## 13.2 dB 转换

电压波 / 场幅类 S 参数：

```text
Sij_dB = 20 * log10(abs(Sij))
```

功率比：

```text
P_dB = 10 * log10(Pout / Pin)
```

注意：

- S 参数通常用 20log10；
- 功率、能量、效率通常用 10log10；
- 不要把功率和场幅公式混用。

## 13.3 导出数据时要保留什么

如果要和 MATLAB 或老板给的 `snp_analyse.m` 对比，导出时最好保留：

```text
频率
S11 实部
S11 虚部
S21 实部
S21 虚部
S31 实部
S31 虚部
S41 实部
S41 虚部
```

或者直接导出标准 Touchstone 文件：

```text
.s2p
.s3p
.s4p
.s5p
```

常见问题：

- 只导出了 dB，无法计算复数误差；
- 没有实部 / 虚部，无法生成标准 SNP；
- 端口数不一致；
- 文件中频率单位不一致；
- WavEDA 和 COMSOL 频点不一致，需要插值。

## 13.4 后处理对比流程

推荐流程：

```text
读取 WavEDA 数据
读取 COMSOL / EastWave 数据
统一频率单位
统一频率采样点
必要时插值
计算 abs(Sij)
转换 dB
计算误差
画对比图
生成误差表
```

误差表建议包含：

```text
模型名称
软件
频率范围
频点数
插值点数
网格数量
DOF
耗时
S11 误差
S21 误差
S31 误差
S41 误差
备注
```

# 14. 常见报错与排查库

## 14.1 报错：后验误差计算时出现错误，请检查激励源是否设置正确

常见原因：

1. 没有设置激励源；
2. 端口方向错；
3. 激励端口没有选中；
4. 端口截面不完整；
5. 端口模式求解失败；
6. 端口与边界冲突；
7. PML 与波端口不兼容；
8. 模型没有形成有效传播路径；
9. 频率单位错，导致模式不存在；
10. 材料设置导致端口模式异常。

排查顺序：

```text
先看端口是否有模式图
    ↓
看端口箭头是否进入计算域
    ↓
确认只激励一个输入端口
    ↓
确认输出端口不激励但参与求解
    ↓
检查端口截面是否覆盖完整波导
    ↓
检查边界是否与端口冲突
    ↓
检查材料和频率单位
    ↓
再改网格
```

Agent 回答模板：

```text
这个报错名字说的是“后验误差”，但根子通常不在误差本身，而在激励源。
你先别改网格，按这个顺序查：

1. 输入端口有没有真正设置为激励；
2. 端口箭头是不是指向计算域；
3. 端口模态能不能正常显示；
4. 输出端口是不是只作为接收端口；
5. 端口面有没有完整覆盖波导截面；
6. 有没有把波端口和 PML 放在同一个边界体系里。
```

## 14.2 报错：波端口与 PML 不兼容

常见原因：

- 显式 PML 覆盖了端口边界；
- 端口面在 PML 区域里；
- 求解器不支持当前波端口 + PML 组合；
- 同一外边界同时被设为端口和吸收层。

处理建议：

1. 删除或禁用端口方向上的 PML；
2. 使用 WavEDA 默认吸收边界；
3. 保证端口面在普通计算区域边界上；
4. PML 只放在非端口开放方向；
5. 重新检查计算域尺寸。

## 14.3 问题：S21 很小，但场图看起来传过去了

优先怀疑：

1. 输出端口方向反了；
2. 读取了错误模式；
3. 读取了错误 S 参数通道；
4. 输出端口截面太小；
5. 端口没有覆盖完整模式；
6. S 参数 dB 公式错；
7. 输出端口编号和预期不一致。

处理模板：

```text
场图能传过去但 S21 很低，说明物理传播未必错，先查“测量口径”。
请你先看端口模态图和 S 参数变量名，确认你读的是 Port1 → Port2 的目标模式，而不是 mode2 或反向通道。
```

## 14.4 问题：S11 很大，几乎全反射

可能原因：

- 输入端口模式不匹配；
- 端口太靠近结构突变；
- 端口方向错；
- 材料错；
- 波导截面尺寸错；
- 边界反射；
- 几何有断裂；
- 光源频率不在导模范围；
- 网格过粗。

排查顺序：

```text
端口模式 → 几何连续性 → 材料 → 边界距离 → 网格 → 频率范围
```

## 14.5 问题：扫频曲线很乱

可能原因：

- 单频没有验证就扫频；
- 网格精度不足；
- 端口模式在扫频过程中跳模；
- 插值点数不够；
- 后处理频率点不一致；
- 频率 / 波长横轴混用；
- 求解未收敛；
- 高 Q 结构对网格敏感。

建议：

```text
回到中心频点单频验证。
再选择低频、中频、高频三个点分别跑，确认场图是否一致。
不要直接盯着整条曲线改参数。
```

## 14.6 问题：端口打不开 / 软件界面异常

历史中出现过端口界面打不开的情况。

处理建议：

1. 保存当前工程副本；
2. 重启 WavEDA；
3. 删除异常端口重新建；
4. 检查几何面是否被修改；
5. 检查导入模型是否有坏面；
6. 尽量不要在异常工程上继续堆设置；
7. 必要时回退到上一个可打开版本。

# 15. WavEDA 与 COMSOL 对比原则

## 15.1 对比前必须统一的口径

```text
几何尺寸
材料参数
背景区域
边界条件
端口位置
端口截面
端口方向
端口模式数
激励端口
频率范围
频率采样点
后处理变量
dB 转换公式
```

任何一项不同，都可能导致曲线差异。

## 15.2 不同软件结果不一致时的排查顺序

推荐优先级：

```text
材料
    ↓
端口 / 光源
    ↓
边界
    ↓
后处理公式
    ↓
网格
    ↓
求解器差异
```

不要一开始就把锅甩给网格。

历史经验：

- 012 模式转换器的主要问题是材料设置，不是网格；
- 03 定向耦合器重点是端口模式数和 S 参数通道；
- 09 光栅耦合器主要差异来自光源口径，WavEDA 中缺少与 COMSOL 完全一致的高斯面光源；
- Ref-A / ref’-A 结构重点是周期 / Floquet / 端口角度。

```text
5. 端口截面是否覆盖完整模式；
6. 模式数是 1 还是 2；
7. 你读取的是不是正确的 S 参数通道。

尤其是光波导，端口不是只选硅芯表面，而是要覆盖完整模场截面。
```

## 16.4 结果不对模板

```text
结果不对先别急着改网格。我们按优先级排：

1. 材料有没有和参考统一；
2. 频率单位是 GHz 还是 THz；
3. 端口 / 光源是不是同一种物理口径；
4. 边界有没有反射或冲突；
5. S 参数通道有没有读错；
6. dB 转换公式有没有错；
7. 最后再考虑网格和求解精度。
```

# 18. 附录 B：09 直栅型光栅耦合器经验模板


6. 输出重点看 `Sx` 或 `-Sx`，而不是盲目看 S21。

# 22. 最重要的经验总结

## 22.1 不要先扫频

全频扫频是最后一步，不是第一步。  
先单频，后扫频。

## 22.2 不要把所有问题都归因于网格

很多历史问题根源不是网格，而是：

- 材料错；
- 端口错；
- 光源错；
- 边界错；
- S 参数通道错；
- 后处理公式错。

## 22.3 端口问题看三件事

```text
端口面
端口方向
端口模式
```

只看 S 参数曲线不够，必须看端口模态和场图。

## 22.4 光栅耦合器不是普通端口网络

光栅耦合器应按：

```text
光源入射 → 光栅耦合 → 输出波导截面功率
```

而不是强行按三端口 S 参数理解。

## 22.5 WavEDA 和 COMSOL 对比要统一口径

如果两边设置不一致，误差没有意义。

必须统一：

```text
材料
几何
边界
端口 / 光源
频率
模式
后处理
```

## 22.6 看结果时先问“物理是否合理”

一条曲线好不好看不是第一标准。  
先看：

```text
场有没有进来
能量有没有传输
输出有没有接收
反射是否合理
端口模式是否正确
```

再看 S 参数曲线。
